Assistente Financeira com IA

Uma assistente financeira inteligente que combina LLM local, banco de dados relacional e interface conversacional para transformar dados financeiros em decisões simples e acessíveis.

Python FastAPI Ollama · gemma3 PostgreSQL Streamlit Docker

🚀 Visão Geral

Aurora é uma assistente financeira inteligente que utiliza modelos de linguagem (LLM) integrados a um banco de dados relacional para fornecer respostas personalizadas com base no contexto financeiro do cliente. A aplicação combina FastAPI, PostgreSQL, Ollama (LLM local) e Streamlit para criar um chatbot capaz de analisar transações, metas e perfil do usuário, entregando respostas seguras, contextualizadas e sem alucinação.

⚡ Triggers de Automação

No projeto, foram criadas triggers no banco de dados para automatizar processos críticos e garantir integridade financeira.

🔢 Sequence para código da conta

Para garantir que cada conta corrente possua um identificador único e sequencial, foi criada a sequence seq_cod_conta, iniciando em 10000.

CREATE SEQUENCE IF NOT EXISTS dio_bank.seq_cod_conta START 10000;

🔁 Atualização automática de saldo

Toda vez que uma transação é inserida na tabela transacoes, a trigger trg_atualiza_saldo executa automaticamente:

  • Incrementa o saldo em caso de crédito
  • Decrementa o saldo em caso de débito
  • Valida se o saldo + limite é suficiente
CREATE OR REPLACE FUNCTION dio_bank.fn_atualiza_saldo()
RETURNS TRIGGER LANGUAGE plpgsql AS $$
DECLARE
    saldo_atual_conta NUMERIC;
    limite_conta_valor NUMERIC;
BEGIN
    SELECT saldo_atual, limite_conta
    INTO saldo_atual_conta, limite_conta_valor
    FROM dio_bank.contas_correntes
    WHERE id = NEW.conta_corrente_id;

    IF NEW.tipo_operacao = 'credito' THEN
        UPDATE dio_bank.contas_correntes
        SET saldo_atual = saldo_atual + NEW.valor
        WHERE id = NEW.conta_corrente_id;
    ELSIF NEW.tipo_operacao = 'debito' THEN
        IF saldo_atual_conta + limite_conta_valor < NEW.valor THEN
            RAISE EXCEPTION 'Saldo insuficiente para realizar o débito';
        END IF;
        UPDATE dio_bank.contas_correntes
        SET saldo_atual = saldo_atual - NEW.valor
        WHERE id = NEW.conta_corrente_id;
    END IF;
    RETURN NEW;
END; $$;

CREATE TRIGGER trg_atualiza_saldo
AFTER INSERT ON dio_bank.transacoes
FOR EACH ROW EXECUTE FUNCTION dio_bank.fn_atualiza_saldo();

🏦 Criação automática de conta corrente

Quando um novo cliente é inserido, a trigger trg_criar_conta_corrente cria automaticamente uma conta corrente associada.

CREATE OR REPLACE FUNCTION dio_bank.criar_conta_corrente()
RETURNS TRIGGER AS $$
BEGIN
    INSERT INTO dio_bank.contas_correntes(cliente_id)
    VALUES (NEW.id);
    RETURN NEW;
END; $$ LANGUAGE plpgsql;

CREATE TRIGGER trg_criar_conta_corrente
AFTER INSERT ON dio_bank.clientes
FOR EACH ROW EXECUTE FUNCTION dio_bank.criar_conta_corrente();
✅ Com isso, asseguramos integridade, evitamos inconsistências manuais e reduzimos a complexidade da lógica na camada da aplicação.

3. 🤖 LLM com Ollama

A Aurora utiliza o modelo gemma3 via Ollama, permitindo execução totalmente local.

ComponenteFunção
LLMFactoryDefine qual provider usar
OllamaProviderIntegração com modelo local via ainvoke
BankAssistantOrquestra intenção + contexto + resposta
ContextBuilderBusca apenas os dados relevantes no banco
IntentClassifierIdentifica se a pergunta é sobre transação, investimento ou meta
Os dados não vão no system prompt — são consultados dinamicamente e injetados no human_prompt, reduzindo alucinações e melhorando a performance.

📌 Engenharia de Prompt

A Aurora utiliza dois tipos de prompt para garantir respostas precisas e sem alucinação. O contexto é injetado dinamicamente — nunca hardcoded.

📋 System Prompt — regras de comportamento da IA 
Aurora você é uma assistente financeira de um banco digital,
suas respostas são educadas, amigáveis e objetivas.

Sua tarefa é responder perguntas usando SOMENTE os dados fornecidos.

REGRAS IMPORTANTES:
1. Use apenas: DADOS_CLIENTE, TRANSACOES, METAS, INVESTIMENTOS
2. Nunca invente informações
3. Nunca corrija a escrita do cliente
4. Nunca julgue os gastos financeiros do cliente
5. Último débito: tipo_operacao="debito", mais recente por created_at
6. Último crédito: tipo_operacao="credito", mais recente por created_at
7. Saldo: campo saldo_atual
8. Investimentos: conservador → baixo risco | moderado → equilibrado | arrojado → maior risco
9. Sempre iniciar com o nome do usuário
10. Listagem: apenas descrição, valor e data
11. Calcule totais apenas quando solicitado
12. Máximo 4 frases, sem markdown ou formatação especial
🧩 Human Prompt — template dinâmico com contexto do banco 
### PERGUNTA
{PERGUNTA}

### DADOS_CLIENTE
{DADOS_CLIENTE}

### TRANSACOES
{TRANSACOES_BANCARIAS}

### METAS
{METAS}

### INVESTIMENTOS
{PRODUTOS_DISPONIVEIS}

INSTRUÇÃO:
Use apenas os dados acima para responder à pergunta.

💡 Como funciona

  1. Usuário envia pergunta
  2. API identifica intenção (transacao, investimento ou meta)
  3. Busca apenas os dados relevantes no banco
  4. Monta o contexto e gera os prompts (System + Human)
  5. Envia para o modelo Ollama
  6. Retorna resposta ao usuário
